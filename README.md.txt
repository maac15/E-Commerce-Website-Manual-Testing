# E-Commerce Website Manual Testing

## Overview
This repository contains all QA activities for a sample E-Commerce website, focusing on manual testing.

## Folder Structure
- **TestCases/** : Excel files for manual test cases (Login, Product Search, Shopping Cart, Checkout)
- **TestReports/** : Execution reports for manual tests
- **Bugs/** : Logged bugs
- **Documentation/** : QA guides and instructions

## How to Use
1. Open relevant test case from `TestCases/` folder.
2. Execute the test manually on the application.
3. Record results in `TestReports/`.
4. Log any bugs in `Bugs/Bug_List.xlsx`.
